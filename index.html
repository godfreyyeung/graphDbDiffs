Difference Category Descriptions

Unless otherwise stated, the following difference descriptions refer to differences of a given claim between one or more databases.
  1. Structural
    Differences related to node and link placement.
    1. Different Triple/Nodes
      The triple statements themselves use one or more different entities or values.
      1. Different Predicate
        The predicate node is a different entity.
      2. Different Object
        The object node is a different entity or value.
        1. Different Entity
          The object node is a different entity.
        2. Different Value
          The object node is a different literal, i.e. a different string or number.
        3. Single Entity vs Value
          The object is an entity in one DB, and a value in another.
        4. Embedded Object
          In one DB a statement's object only consists of one focused, single entity or value. In another DB that that object consists of multiple additional entities or values embedded alongside the one entity or value. Requires the predicates to be equivalent or conceptually serve the same purpose.
          1. Value
            In one DB, a statement's object is a single value, and in another that single value is embedded into part of a larger value.
          2. Object
            In one DB, astatement's object is a single entity, and in another that entity conceptually represents multiple other resources.
        5. Multipart Object
          One DB represents a resource as a single entity or value, while another DB uses multiple statements to represent that resource as multiple entities or value. This may mean that one DB conceptually represents an entity as having one part, while another DB represents that entity as comprised of multiple parts. Does not require the predicates to be equivalent, just that conceptually the multiple statements one database serve the same purpose as the single statement in the other.
      3. Both Predicate and Object are Different
        The given example exhibits both differences 1ai and 1aii.
    2. Extended Statement
      Compared to a given statement forming a claim in one database, a statement forming the same claim in another database has additional intermediate statements before or after the predicate of the given claim.
      1. General case
        The general case assumes the subject, predicate and object are the equivalent across the compared DBs.
      2. Added Expressivity
        The additional intermediate statements add expressiveness to the claim.
      3. Encapsulated Object
        The additional intermediate statements encapsulates the original object, which may be either an entity or a value, without adding any expressivity to the claim. The new intermediate entity may only exist to represent the original entity at a higher level, holding the original entity as a property.
  2. Value
    Differences regarding values.
    1. Unit of Measure attached to different nodes
      For example, the currency unit is attached to the predicate in one DB, while to the object in another DB.
    2. Differing underlying method of calculation
      Numeric values have different assumed underlying methods of calculation.
  3. Meta
    Differences stem from conceptual differences between databases, such as how they view the world.
    1. Claim inferrable from another claim
      Within one database, a given claim can be implied or inferred from statements meant to make another claim. That database may or may not have statements explicitly forming the given claim.
    2. Multiple different statements make the same claim
      Within one database there are multiple statements explicitly forming the same claim.
    3. Different view of the world
      1. Nuance expressed with additional statements
        Nuanced view is created using additional statements, which are not necessarily “extended statements”
      2. Predicate considered an Object, or vice versa
        An entity is used as a predicate in one DB but as an object in another.
      3. Different idea of entity
        Given equivalent entities across databases (e.g. perhaps linked using owl:sameAs) different databases may still have different ideas on what that entity inherently means or how it should be used.
      4. Different entity label for same resource
        Different databases may label an entity differently, but the underlying resource of the entities are the same.

Instructions

This tool visualizes triples (statements) that make up a claim. It also categorizes example claims by the cross-database differences exhibited their triples.
  1. Click on an example claim under the "Examples" tab to load graph visualizations its triples.
  2. Click the "View Categories" button to filter examples by difference category. An example only appears if it is associated with all checkboxed categories. Scroll down on the categories pane to see descriptions of each category.
  3. Once an example is selected, click the "Example Description" tab to read about the differences exhibited by that example. All exhibited differences are also shown in a list, bolded and highlighted.
  4. Zoom out on your browser (Ctrl, -) for the best viewing experience of the graph visualizations. Use Chrome or Firefox for best compatability.
  5. In the graphs, blue nodes are subejcts and objects, while orange nodes are predicates
    6. .
  6. Thus, a triple (statement) consists of a blue, orange, and another blue node, chained in that order. Orange nodes should always sit between blue nodes.
  7. The very first statement in a visualization is laid horizontally.
  8. A given statement's links will be the same color. The link colors change depending on the number of statements that sit between the statement's subject and the first (horizontal) statement. For example, if a statement's subject is the subject, predicate or object of the first statement, then its links will be the same color as the first statement. Statements whose subject node is the predicate or object of this latter statement will be another color, and so on.
  9. Click and drag nodes to re-adjust and untangle the graphs. The graph is not smart enough to position the nodes perfectly. For example, for many graphs the nodes must be dragged so that blue nodes on each side of an orange node, instead of the orange node sitting to the left or right of two blue nodes.
  10. Click "Toggle Overlaps" to prevent nodes from overlapping one another. Click again to deactivate this feature.
  11. If a graph is too hard to untangle, click "Show Image" (on complex graphs) to view an image of the untangled graph.
  12. Use the range slider at the bottom left of each graph to adjust link length.
  13. Click "Show triples" to see the original triples that make up the visualized graph.
  14. Literal values are wrapped in single quotes. For example '$342'.
  15. The statements shown are hand selected. They may or may not be relevant to the differences exhibited.

Glossary

Most fundamental definitions listed first.

Resource/Item
an abstract concept. A “thing” in the universe.
Entity
A resource’s digital manifestation, meaning its serialized version that has an associated identifying URI or URL.
Value
As opposed to an Entity, a value is an serialized resource without an associated URI.
Subject
An entity being described (by a RDF statement). It is the entity represented by the URI/URL used as the first component in a RDF statement.
Predicate/Property
An entity (used in a RDF statement) describing the relationship between a subject and an object, specifically how the subject is described by the object. It is represented by the URI/URL in the second component of the RDF statement.
Object
An entity (used in a RDF statement) that describes the subject according to some relationship (specified by a predicate). It is represented by the URI/URL in the third component of the RDF statement. It can also be a value.
RDF statement/statement/RDF Triple
The concatenation, by some syntax, of subject, predicate, and object URI/URLs to describe a knowledge graph relationship between those entities.
Claim
An assertion of a fact in the universe, made using one or more statements.
Node
A polygon, within a visual representation of a graph database, representing a subject, predicate or object entity within the knowledge database.
Link
A line, within a visual representation of a graph database, representing a predicate entity.
Class
An entity representing a classification of entities into groups which share common characteristics.
Individual/Instance
An entity which is part of a group. The classification of this entity is carried out by a statement describing the entity, i.e. the entity is the subject, with an rdf:type or equivalent predicate and a class object representing the group that the entity is part of.
Subclass
A class entity that is an extension of another class, a parent class. This extension is specified by the instancing of this entity to that parent class.
NS
Namespace
DB
Database
